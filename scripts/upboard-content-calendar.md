### 投稿フォーマット別ガイドライン

**朝7時投稿（体験談型）**
- 文字数: 140〜200字
- 構成: 【タグ】→属性紹介→ストーリー要約→UpBoard言及→診断URL→ハッシュタグ

- 文字数: 150〜220字
- 構成: タイトル（疑問形or数字）→リスト形式本文→CTA→診断URL→ハッシュタグ
- 必須ハッシュタグ: #エンジニア転職 #学歴不問 #上場企業 #UpBoard

**夜20時投稿（実績報告型）**
- 文字数: 120〜180字
- 構成: 実績数値→詳細→問いかけ→診断URL→ハッシュタグ
- 必須ハッシュタグ: #UpBoard #転職 #エンジニア

- 月〜水: ストーリー系（保存されやすい）
- 木〜金: ノウハウ系（シェアされやすい）
- 土〜日: 共感系（いいねされやすい）
- リプライ対応: 投稿後60分以内に返信来たら必ず対応

## 6. スクリプト実行方法

### 事前準備

```bash
# Claude APIキーを環境変数に設定
export ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxxxxxx"

# プロジェクトディレクトリに移動
cd /Users/wantan/AZABU+Project
```

### SEO記事生成スクリプト

```bash
# ランダムキーワードで記事生成（引数なし）
node scripts/generate-upboard-seo.js

# キーワード指定で生成
node scripts/generate-upboard-seo.js "高卒 上場企業 エンジニア 就職"

# 複数キーワードを一括生成する場合（bashループ）
for keyword in "高卒 エンジニア なれる 方法" "学歴フィルター 回避 転職 エンジニア" "専門卒 エンジニア 転職 上場企業"; do
  node scripts/generate-upboard-seo.js "$keyword"
  sleep 5
done

# 出力先: scripts/generated-articles/YYYY-MM-DD-{slug}.md
```

**記事公開フロー（WordPress）**:

```bash
# wp-cli でドラフト投稿（要: wp-cli インストール済み）
wp post create \
  --post_title="高卒が上場企業エンジニアになれる方法【体験談あり】" \
  --post_content="$(cat scripts/generated-articles/2026-04-17-xxxx.md)" \
  --post_status=draft \
  --post_author=1

# WordPress REST API でも可
curl -X POST https://azabuplus.jp/wp-json/wp/v2/posts \
  -H "Authorization: Basic $(echo -n 'user:app-password' | base64)" \
  -H "Content-Type: application/json" \
  -d '{"title":"記事タイトル","content":"記事本文","status":"draft"}'
```

### Xコンテンツ生成スクリプト

```bash
# 7日分生成（デフォルト）
node scripts/upboard-x-content.js

# 30日分生成
node scripts/upboard-x-content.js 30

# 出力先: scripts/x-content/YYYY-MM-DD-x-posts.md
```

**生成ファイルの使い方**:
1. `scripts/x-content/` に日付付きMarkdownで保存される
2. Buffer / Hootsuite で手動コピペ予約投稿、または
3. n8n で自動化（次セクション参照）

## 7. n8n自動化セットアップガイド

### 概要

n8nはノーコードの自動化ツール。ローカルまたはn8n Cloudで動作する。  
公式: https://n8n.io

### インストール（Docker）

```bash
# n8n をローカルで起動
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n

# ブラウザで http://localhost:5678 を開く
```

### フロー1: SEO記事の定期自動生成

```
[スケジュールトリガー: 月・水・金 09:00]
    ↓
[Codeノード: キーワードリストからランダム選択]
    ↓
[HTTPリクエストノード: Claude API呼び出し]
    POST https://api.anthropic.com/v1/messages
    Headers: x-api-key, anthropic-version
    Body: { model, max_tokens, messages }
    ↓
[Codeノード: Markdownファイル生成・フロントマター付与]
    ↓
[FTPノード or GitHubノード: ファイル保存]
    ↓
[GmailノードorSlackノード: 完了通知送信]
```

**設定値**:
- Claude モデル: `claude-sonnet-4-6`
- max_tokens: `4000`
- 環境変数: n8n の Credentials に `ANTHROPIC_API_KEY` を登録

### フロー2: X投稿の自動スケジュール

```
[スケジュールトリガー: 毎日 06:50 / 11:50 / 19:50]
    ↓
[Codeノード: 今日の投稿内容をキューから取得]
    ↓
[HTTPリクエストノード: X API v2 でツイート投稿]
    POST https://api.twitter.com/2/tweets
    OAuth 2.0 認証
    ↓
[Googleスプレッドシートノード: 投稿ログ記録]
```

**X API設定**:
- Developer Portal: https://developer.twitter.com
- 必要権限: Read and Write
- n8n Credentials に Bearer Token と OAuth 1.0a キーを登録

### フロー3: 記事→WordPress自動投稿

```
[Webhookトリガー: ファイル保存完了イベントを受信]
    ↓
[Codeノード: Markdownをパース、タイトル・本文抽出]
    ↓
[HTTPリクエストノード: WordPress REST API]
    POST https://azabuplus.jp/wp-json/wp/v2/posts
    Basic認証（アプリパスワード使用）
    ↓
[Slackノード: 担当者へ確認依頼送信]
```

### n8nワークフローのエクスポート・バックアップ

```bash
# n8n CLI でエクスポート
n8n export:workflow --all --output=./n8n-backup/

# 定期バックアップ（cron）
0 2 * * * n8n export:workflow --all --output=/backup/n8n-$(date +%Y%m%d)/
```

## 10. よくあるトラブルと対処法

### APIエラーが発生する

```bash
# APIキーの確認
echo $ANTHROPIC_API_KEY

# レート制限の確認（429エラーの場合）
# -> sleep 10 を挟んでリトライ
sleep 10 && node scripts/generate-upboard-seo.js "キーワード"
```

### 生成記事の品質が低い場合

- `generate-upboard-seo.js` のプロンプト内の【記事の設定】を更新
- ターゲット読者の年齢・背景をより具体的に記述
- 成功した記事例を few-shot として追加

### X API制限に引っかかる場合
