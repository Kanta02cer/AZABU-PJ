# Google AdSense 配置設計書（CRO × PV 両立）

最終更新: 2026-04-04  
目的: **広告でPV・収益を伸ばしつつ**、本文・CTAを邪魔せず**離脱を抑え、コンバージョン（面談予約）を最大化**する。

---

## 1. 原則（必ず守る）

| 原則 | 内容 |
|------|------|
| **CVファースト** | 面談予約・ブランドハブ・主要CTAの**直上・直後**にディスプレイ広告を置かない |
| **読了優先** | 本文の「最初のスクロール」で広告が主役にならないよう、**視認後に遅延ロード** |
| **密度の上限** | 一覧は **N件に1回** のインフィードに抑え、バナーの積み上げを避ける |
| **一貫ラベル** | 枠に **「スポンサー」** 等の表示で広告領域を明示（体験とポリシー配慮） |
| **計測** | `cv_click` 前後のセッションで広告あり版のCVR変化を比較（GTM/GA4） |

---

## 2. ページタイプ別マップ

### 2.1 記事（`ArticleTemplate` / `/_post/*`）

| スロットID（設定キー） | 配置 | フォーマット | CRO方針 |
|------------------------|------|--------------|---------|
| `articleTop` | ~~ブランドハブ直下〜本文前~~ **既定ではオフ** | 横長 | CTA直後の読み始めを阻害しないため非表示 |
| `articleBottom` | **本文（prose）終了直後** | レスポンシブ | 読了直後＝自然な区切り。シェアの前 |
| `articleInContent` | **関連記事ブロックの下** | レスポンシブ | 回遊前に1枠。記事内の連打感を避ける |
| `sidebar` | **TOC下（デスクトップのみ）** | 300×250 | 本文と競合しにくいサイド |

**推奨読者動線**  
タグ → **ブランドハブ＋面談CTA** → 本文 → （広告・本文下）→ シェア → 前後記事 → 関連記事 → （広告）→ AZABU+PRESS 等

### 2.2 記事一覧（`/azabu-press`）

| スロット | 配置 | 方針 |
|----------|------|------|
| `listingFeed` | グリッド内 **9件ごと**（1行目の優先カルーセルは広告なし） | スクロール中の離脱を抑えるため **6→9** に緩和 |

### 2.3 タグ一覧（`/tag/:tag`）

| スロット | 配置 | 方針 |
|----------|------|------|
| `listingFeed` | **9件ごと** | 一覧と同じ密度 |

### 2.4 高CVページ（非掲載推奨）

| ページ | 広告 |
|--------|------|
| `/` ホーム | **現状どおり非掲載**（ヒーロー・面談導線を優先） |
| `/azabu-plus-tenshoku` 等ブランドハブ | **非掲載推奨**（ブランド検索着地の迷い防止） |
| `/login` 等コンバージョン直前 | **非掲載** |

### 2.5 将来拡張（設計のみ）

- **長文のみミッドロール**: 読了時間 ≥ 8分 かつ 見出しが N 個以上のとき、**最初の h2 の後**に1枠（要: コンテンツ構造のマークアップ対応）
- **アンカー広告**: 記事末「もっと見る」の下に **1枠**（回遊強化）
- **Auto ads**: 方針決定後にのみ検討（レイアウト制御が弱いため初期はオフ推奨）

## 3. 技術方針

| 項目 | 内容 |
|------|------|
| **遅延表示** | `IntersectionObserver`（`rootMargin` 約 320px）でビューポート付近まで **マウントしない** |
| **CLS対策** | ラッパーに `min-height` を付与し、枠の高さを事前確保 |
| **スロット管理** | `src/config/adsense.ts` に集約。本番は `VITE_ADSENSE_SLOT_*` で上書き |
| **A/B** | `VITE_ADSENSE_ENABLE_ARTICLE_TOP` で「本文前バナー」を試験的にオン可能 |

---

## 4. KPI（四半期）

- 記事 **セッションあたりPV**（内部リンク＋関連）
- **広告表示回数 / 1000セッション**
- **cv_click 率**（`button_location` 別、広告導入前後）
- **直帰率**（記事ランディング）

## 5. 運用チェックリスト

- [ ] AdSense でユニットタイプごとにスロット作成 → `.env` に実IDを設定  
- [ ] `ads.txt` が配信ドメインで有効  
- [ ] モバイルで **CTA→本文** の間に広告が入っていないこと  
- [ ] PageSpeed（LCP/CLS）をデプロイ前後で比較
